Past AGI. Toward cognitive systems.

Most frontier labs are racing to build a bigger brain. We're building everything around it — the memory, perception, scope, evidence, and feedback loops that turn a powerful model into a genuinely cognitive system. This is the external cognitive infrastructure of agentic intelligence.

00 · Position

We are not building the brain. We are building everything around it.

The dominant story about AGI is a story about model capability. Bigger transformers, longer contexts, deeper reasoning, multimodal understanding, autonomous planning. That story matters — but it is incomplete.

EloAI's bet is structural. Intelligence is not just the model. Intelligence is the loop — perception, memory, compression, reasoning, action, feedback, correction. A frontier model dropped into a chaotic environment without that loop is a brilliant savant with anterograde amnesia and no hands. Useful, sometimes; reliable, rarely.

Our work is to build the external cognitive infrastructure that lets AI agents operate with memory, context, evidence, scope control, semantic compression, and human-aligned feedback. This makes current models more reliable and creates a foundation that future AGI-level systems would also require. — Position statement · EloAI Research

Frontier labs frame AGI as broadly capable, general, and increasingly autonomous. We accept that framing — and then ask the question those labs largely defer: what is the operating environment a generally capable model would actually need to be trusted with consequential work?

01 · The cognitive loop

Intelligence is a loop, not a layer.

The conventional stack treats the model as the apex and everything else as plumbing. We invert that. The loop is the unit of cognition; the model is one stage of it. Cut any link and the system degrades to a chatbot — fluent, stateless, and unaccountable.

Loop · cognitive cycle
Perception → Memory → Compression → Reasoning → Action → Feedback → Correction ↺
A model without this loop is a stateless oracle. A model wired into it becomes continuing intelligence — capable of learning from consequence rather than restarting every conversation.

Every active program in the lab is a deliberate intervention in one or more links of this loop. Treat the table below as a map of our bets — which AGI-relevant capability each program is trying to unlock.

02 · Capability map

How our experiments project onto AGI-relevant capabilities.

DeepMind's Levels of AGI framework treats AGI as a combination of performance, generality, and autonomy. OpenAI frames it as systems generally smarter than humans, deployed safely. Both definitions share the same omission: a clear account of where the agent's competence actually lives. Inside the weights? In the surrounding system? Across the loop?

We argue most of it lives in the loop. Here is how our programs map onto the capability surface usually attributed to "AGI":

03 · Strongest AGI-aligned bets

Five experiments doing real work on the loop.

The full program portfolio is broader, but these five are the most AGI-relevant — each one targets a link in the loop that today's frontier models cannot supply for themselves.

01

Memory and recallMEM-α

Today's models are fragmented along memory: user-controlled, opaque, or window-bound. A self-directed memory subsystem lets an agent decide what future context is worth preserving — and what to discard. This is the difference between a stateless assistant and a continuing intelligence that accrues identity across sessions.

02

AIS-RTS scope intelligenceATL-02

One of the most practical interventions in the lab. AIS-RTS gives an agent a structured model of a software project — symbols, files, imports, relationships, schemas, endpoints, configs — without stuffing the repository into the context window. Before any edit, the agent already knows the impact zone. Hallucinated structure goes from a routine failure mode to a checkable, falsifiable claim.

03

Semantic compressionLAT-04

Context is expensive, noisy, and lossy. Our compressed index work — LSDF-inspired graph representations of structure, meaning, symbols, and relationships — produces compact references the agent can carry across turns and tasks. A better working memory layer for the loop, measured in tokens saved and recall fidelity preserved.

04

Meaning matrix · resonance graphNOE-03

The most speculative bet — and the closest to a cognitive architecture in the classical sense. The resonance graph tracks live state across a session: intent, topic, action, relationship, semantic pressure. A lightweight perception system for language. We expect most of its value to surface as negative capabilities — the ability to notice when something is off.

05

TruthEngine · DAISCUMB-05

Grounding as a first-class subsystem. Source maps, evidence trails, claim verification, contradiction detection, bias surfacing, provenance. Frontier labs treat interpretability and alignment as foundations for safe AI. We treat them as product-grade data plumbing — usable on day one, not after a paradigm shift.

04 · The gap

What we don't yet have: closed-loop autonomous learning.

We are honest about the missing piece. Today our systems help agents understand context, retrieve memory, inspect structure, plan better, verify claims, and operate more safely. They do not yet, as a unified architecture, learn from their own experience.

What our loop supplies today
  • Context perception & project scope
  • Self-directed memory writes & recall
  • Compressed semantic indices
  • Bounded tool execution
  • Claim grounding & provenance
  • Human-in-the-loop correction channels
What full cognition still needs
  • Goal formation under uncertainty
  • Outcome measurement against intent
  • Failure-cause attribution
  • Strategy updates from experience
  • Performance improvement over time
  • Cross-domain transfer of lessons

The bridge between the two columns is a single artifact: a persistent record of what the agent tried, why it tried it, and what happened. We call it the Agent Learning Ledger.

05 · Agent learning ledger

From context-aware to experience-aware.

The ledger is the next deliberate intervention in the loop. It is not a training run, a reward model, or a fine-tune. It is a structured journal — readable by the agent, auditable by the human — that turns each completed task into a row of compressed experience the agent can consult next time.

Field
What it records
task.attempted
What the user actually wanted, in the agent's own words after clarification.
plan.chosen
The strategy the agent committed to, and the alternatives it considered.
tools.invoked
Which tools were used, in what order, with what arguments.
outcome
Success, partial success, or failure — judged against the original intent.
error.type
Missing file · bad assumption · out-of-scope edit · weak prompt · bad retrieval.
correction
What ultimately fixed it — human nudge, tool retry, or self-revision.
lesson.reusable
A compressed maxim the agent should retrieve when it sees a similar shape next time.

Once the ledger is queryable through the same recall subsystem the agent uses for memory, the loop closes. Every task becomes potential training data for every future task — without retraining a single weight.

06 · Strategic frame

The lane.

A more powerful brain without a reliable operating environment is still chaotic. Smarter models will still need memory, scope, truth, tools, permissions, source maps, and feedback. The infrastructure we are building becomes more valuable as foundation models improve, not less.
